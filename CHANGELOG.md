## [0.1.0-next.1](https://github.com/radio-garden/react-native-audio-browser/compare/v0.0.0...v0.1.0-next.1) (2026-03-20)

### ⚠ BREAKING CHANGES

* **ios:** isIosOutputExternal() replaced with getIosOutput(),
useIosOutputExternal() replaced with useIosOutput()
* metadata callback names and parameter types changed
* capabilities API changed from array to object
* **ts:** useDebug signature changed from (enabled: boolean)
to (options: DebugOptions) and now returns { state, logs, clear }.
useDebugMetadata is removed - use useDebug({ metadata: true }) instead.
* `play` property renamed to `singleTrack` with
inverted boolean semantics (play:'single' -> singleTrack:true)
* **android:** android.notificationCapabilities replaced with
android.notificationButtons
* `set-rating` capability renamed to `favorite`
* `handleRemoteSetRating` handler removed, use `onRemoteSetRating` listener instead
* Removed the `playable` optional boolean field from Track and ResolvedTrack interfaces. Playability is now determined solely by the presence of the `src` field.

Changes:
- Remove `playable?: boolean` from TypeScript Track and ResolvedTrack interfaces
- Simplify playability logic: tracks are playable if and only if `src != null`
- Update Media3 metadata flags to use `src == null` for browsability checks
- Remove helper functions `isBrowsable()` and `isPlayable()` from TrackFactory
- Update error messages to reflect new validation rules
- Regenerate all Nitro bridge code (C++, Swift, Kotlin)

Migration: Replace `playable: true` with an actual `src` value pointing to the media resource.
* implement navigatePath/navigateTrack split and ResolvedTrack model
* RequestConfig query and headers now only accept string values, HttpMethod enum replaces method strings
* acquireWakeLock() and abandonWakeLock() functions removed, NitroUpdateOptions renamed to NativeUpdateOptions

### ⚠ BREAKING CHANGES

* **ios:** isIosOutputExternal() replaced with getIosOutput(),
useIosOutputExternal() replaced with useIosOutput()

* capabilities API changed from array to object
* **ts:** useDebug signature changed from (enabled: boolean)

* `play` property renamed to `singleTrack` with
inverted boolean semantics (play:'single' -> singleTrack:true)

android.notificationButtons

Changes:
- Remove `playable?: boolean` from TypeScript Track and ResolvedTrack interfaces
- Simplify playability logic: tracks are playable if and only if `src != null`
- Update Media3 metadata flags to use `src == null` for browsability checks
- Remove helper functions `isBrowsable()` and `isPlayable()` from TrackFactory
- Update error messages to reflect new validation rules
- Regenerate all Nitro bridge code (C++, Swift, Kotlin)

Migration: Replace `playable: true` with an actual `src` value pointing to the media resource.
* implement navigatePath/navigateTrack split and ResolvedTrack model
* RequestConfig query and headers now only accept string values, HttpMethod enum replaces method strings
* acquireWakeLock() and abandonWakeLock() functions removed, NitroUpdateOptions renamed to NativeUpdateOptions
